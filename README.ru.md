# shikimori-manager

> [🇬🇧 English](README.md) | 🇷🇺 Русский

Консольная и веб-утилита, которая переносит **аниме и мангу** между списками на [Шикимори](https://shikimori.one) по настраиваемым правилам — например, «всё из *Запланировано*, что уже начато → *Смотрю*» или «высоко оценённые из *Брошено* → *Отложено*».

> ⚠️ Проект не связан с Шикимори. Используйте на свой риск; всегда проверяйте предпросмотр перед применением.

## Возможности

- **Аниме и манга**: одни и те же правила работают для обоих — серии для аниме, главы для манги
- **На правилах**: переносы описываются как «откуда → куда» с необязательными условиями по прогрессу, личной оценке и общему баллу
- **Веб-интерфейс** (Vue 3): переключение языка RU/EN, форма ввода данных, авторизация в один клик, статистика, экспорт в JSON и живой прогресс-бар
- **Безопасно по умолчанию**: предпросмотр ничего не меняет, пока вы не нажмёте «Применить»
- **Вежливо к API**: соблюдает лимиты Шикимори, баллы запрашивает пачками
- **Авторизация «из коробки»**: OAuth один раз, токен кэшируется и обновляется сам

## Установка

```bash
git clone https://github.com/your-username/shikimori-manager
cd shikimori-manager

# только CLI:
python -m pip install -e .

# CLI + веб-интерфейс:
python -m pip install -e ".[web]"
```

Команда устанавливается как `shikimori-manager` (то же самое — `python -m shikimori_manager`).

## Настройка

### 1. Зарегистрируйте OAuth-приложение на Шикимори

1. Откройте <https://shikimori.one/oauth/applications> и создайте приложение.
2. В **Redirect URI** укажите `urn:ietf:wg:oauth:2.0:oob`.
3. В **Scopes** отметьте `user_rates` (и `user`).
4. Скопируйте **Client ID** и **Client Secret**.

### 2. Укажите учётные данные

```bash
cp config.example.toml config.toml   # впишите client_id / client_secret
shikimori-manager auth                # открыть URL, вставить код (один раз)
```

Вместо файла конфигурации можно использовать переменные окружения:

| Переменная | Описание |
|---|---|
| `SHIKI_CLIENT_ID` | Client ID OAuth-приложения |
| `SHIKI_CLIENT_SECRET` | Client Secret OAuth-приложения |
| `SHIKI_USER` | Имя пользователя на Шикимори |
| `SHIKI_USER_AGENT` | Произвольная строка User-Agent |

## Использование

### Веб-интерфейс (рекомендуется)

```bash
# Собрать фронтенд один раз
cd web && npm install && npm run build && cd ..

# Запустить сервер
shikimori-manager serve   # http://127.0.0.1:8000
```

1. Вкладка **«Настройки и вход»** → введите Client ID и Client Secret → *Сохранить настройки*
2. Нажмите **«Открыть страницу авторизации»**, подтвердите доступ, вставьте одноразовый код → **«Отправить код»**
3. Вкладка **«Обзор»** → *Загрузить статистику* или *Экспорт всего в JSON*
4. Вкладка **«Перенос по правилам»** → добавьте правила или возьмите пресет → **«Предпросмотр»** → **«Применить»**

Для разработки фронтенда с горячей перезагрузкой:

```bash
shikimori-manager serve    # терминал 1 — API на :8000
cd web && npm run dev      # терминал 2 → http://localhost:5173
```

### Командная строка (CLI)

```bash
shikimori-manager whoami                   # проверить авторизацию
shikimori-manager lists                    # сколько записей в каждом списке
shikimori-manager stats                    # количество / серии / средний балл
shikimori-manager export -o backup.json    # бэкап всех списков в JSON
shikimori-manager run                      # предпросмотр переносов
shikimori-manager run --apply              # выполнить переносы
```

Частые флаги `run`:

| Флаг | Значение |
|---|---|
| `-c, --config PATH` | Другой файл конфигурации (по умолчанию `config.toml`) |
| `--apply` | Выполнить переносы (без флага — предпросмотр) |
| `--verbose` | Показать каждый перенос, а не только выборку |
| `--sample N` | Сколько переносов показать в предпросмотре (по умолчанию 20) |

## Как писать правила

Правило переносит записи из списка «откуда» (`source`) в список «куда» (`target`) для заданного типа `media` (**аниме** или **манга**), когда выполняются его условия. Правила проверяются **сверху вниз по каждой паре (media, source)** — срабатывает **первое подходящее**, поэтому порядок важен.

Статусы: `planned` (запланировано), `watching` (смотрю/читаю), `rewatching` (пересматриваю), `completed` (просмотрено), `on_hold` (отложено), `dropped` (брошено).

```toml
[[rules]]
name = "started -> watching"   # необязательная метка
media = "anime"                # "anime" (по умолчанию) или "manga"
source = "planned"             # обязательно
target = "watching"            # обязательно
min_episodes = 1               # условия необязательны
```

### Условия

Все условия необязательны и **включительны** (границы входят).

| Условие | Ключи | Псевдонимы | Значение |
|---|---|---|---|
| Прогресс | `min_progress` / `max_progress` | `min_episodes`, `min_chapters`, … | Просмотрено серий или прочитано глав |
| Личная оценка | `min_score` / `max_score` | — | Ваша оценка (0 = без оценки) |
| Общий балл | `min_rating` / `max_rating` | `min_anime_score`, `min_manga_score`, … | Общий балл тайтла на Шикимори |

Баллы дробные (например, `8.38`). Чтобы выразить *строго больше 8*, используйте `min_rating = 8.01`.

### Примеры

```toml
# Аниме «Запланировано»: начатые → смотрю; нетронутые с высоким баллом → отложено
[[rules]]
media = "anime"
source = "planned"
target = "watching"
min_episodes = 1

[[rules]]
media = "anime"
source = "planned"
target = "on_hold"
max_episodes = 0
min_rating = 8.01

# Манга «Брошено»: непрочитанные с высоким баллом → отложено; остальные → запланировано
[[rules]]
media = "manga"
source = "dropped"
target = "on_hold"
max_score = 0
max_chapters = 0
min_rating = 8.01

[[rules]]
media = "manga"
source = "dropped"
target = "planned"
max_score = 0
max_chapters = 0
```
