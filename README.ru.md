# shikimori-manager

*Язык: [English](README.md) · **Русский***

Консольная **и веб**-утилита, которая переносит **аниме и мангу** между списками
на [Шикимори](https://shikimori.one) по **настраиваемым правилам** — например,
«всё из *Запланировано*, что уже начато → *Смотрю*» или «высоко оценённые из
*Брошено* → *Отложено*».

- **Аниме и манга**: одни и те же правила работают для обоих (структура на
  Шикимори одинаковая — серии для аниме, главы для манги).
- **На правилах**: переносы описываются как «откуда → куда» с необязательными
  условиями по прогрессу, вашей личной оценке и общему баллу тайтла.
- **Веб-интерфейс** (Vue 3): переключение языка RU/EN, форма ввода данных,
  авторизация в один клик, статистика по спискам, экспорт в JSON и перенос по
  правилам с живым **прогресс-баром**.
- **Безопасно по умолчанию**: предпросмотр ничего не меняет, пока вы не нажмёте
  «Применить».
- **Вежливо к API**: соблюдает лимиты Шикимори, баллы запрашивает пачками.
- **Авторизация «из коробки»**: OAuth один раз, токен кэшируется и обновляется
  сам.

> ⚠️ Проект не связан с Шикимори. Используйте на свой риск; всегда проверяйте
> предпросмотр перед применением.

## Требования

- Python 3.9+
- Node.js 18+ (только если нужен веб-интерфейс)
- Аккаунт на Шикимори

---

## 1. Зарегистрируйте OAuth-приложение на Шикимори

1. Откройте <https://shikimori.one/oauth/applications> и создайте приложение.
2. В **Redirect URI** укажите `urn:ietf:wg:oauth:2.0:oob`.
3. В **Scopes** отметьте `user_rates` (и `user`).
4. Скопируйте **Client ID** и **Client Secret** — они понадобятся дальше.

---

## 2. Установка

```bash
git clone https://github.com/your-username/shikimori-manager
cd shikimori-manager

# только CLI:
python -m pip install -e .

# ...или CLI + веб-интерфейс:
python -m pip install -e ".[web]"
```

Команда устанавливается как `shikimori-manager` (то же самое —
`python -m shikimori_manager`).

---

## 3а. Веб-интерфейс (рекомендуется)

```bash
# собрать фронтенд один раз
cd web && npm install && npm run build && cd ..

# запустить сервер
shikimori-manager serve            # http://127.0.0.1:8000
```

Дальше в браузере:

1. При желании переключите язык кнопкой **RU/EN** (вверху справа).
2. Вкладка **«Настройки и вход»** → введите **Client ID** и **Client Secret** →
   *Сохранить настройки*. (Хранится в gitignore-файле `.env`.)
3. Нажмите **«Открыть страницу авторизации»**, подтвердите доступ, скопируйте
   одноразовый код (живёт ~2 минуты), вставьте его и нажмите **«Отправить код»**.
   Токен сохранится в `.token.json` и далее будет обновляться автоматически.
4. Вкладка **«Обзор»** → *Загрузить статистику* (количество, серии, средний
   балл) или *Экспорт всего в JSON* для бэкапа.
5. Вкладка **«Перенос по правилам»** → добавьте правила или возьмите пресет,
   **«Предпросмотр»** покажет, что будет перенесено (с прогресс-баром), затем
   **«Применить»**.

### Разработка фронтенда (hot reload)

```bash
shikimori-manager serve            # терминал 1 (API на :8000)
cd web && npm run dev            # терминал 2 -> http://localhost:5173
```

Сервер по умолчанию слушает только `127.0.0.1` — это локальный инструмент, и
выставлять его в сеть не нужно.

## 3б. Командная строка (CLI)

```bash
cp config.example.toml config.toml      # затем впишите client_id / client_secret
shikimori-manager auth                     # открыть URL, вставить код (один раз)

shikimori-manager whoami                   # проверить авторизацию
shikimori-manager lists                    # сколько записей в каждом списке
shikimori-manager stats                    # количество / серии / средний балл
shikimori-manager export -o backup.json    # бэкап всех списков в JSON
shikimori-manager run                      # предпросмотр переносов
shikimori-manager run --apply              # выполнить переносы
```

Данные можно задать и через переменные окружения вместо файла конфигурации:
`SHIKI_CLIENT_ID`, `SHIKI_CLIENT_SECRET`, `SHIKI_USER`, `SHIKI_USER_AGENT`.

Частые флаги `run`:

| Флаг | Значение |
| --- | --- |
| `-c, --config PATH` | Другой файл конфигурации (по умолчанию `config.toml`). |
| `--apply` | Выполнить переносы (без флага — предпросмотр). |
| `--verbose` | Показать каждый перенос, а не только выборку. |
| `--sample N` | Сколько переносов показать в предпросмотре (по умолчанию 20). |

## Как писать правила

Правило переносит записи из списка «откуда» (`source`) в список «куда»
(`target`) для заданного типа `media` (**аниме** или **манга**), когда
выполняются его условия. Правила проверяются **сверху вниз по каждой паре
(media, source)**, срабатывает **первое подходящее** — поэтому порядок важен.

Аниме и манга на Шикимори устроены одинаково, поэтому условия общие — отличается
только `media`.

В CLI правила лежат в `config.toml`:

```toml
[[rules]]
name = "started -> watching"   # необязательная метка
media = "anime"                # "anime" (по умолчанию) или "manga"
source = "planned"             # обязательно
target = "watching"            # обязательно
min_episodes = 1               # условия ниже — необязательны
```

В веб-интерфейсе те же правила собираются в форме (есть выбор типа и два пресета).

Статусы: `planned` (запланировано), `watching` (смотрю/читаю), `rewatching`
(пересматриваю), `completed` (просмотрено), `on_hold` (отложено), `dropped`
(брошено).

### Условия

Все условия необязательны и **включительны** (границы входят). Они медиа-нейтральны,
с удобными псевдонимами.

| Условие | Канонические ключи | Псевдонимы | Значение |
| --- | --- | --- | --- |
| Прогресс | `min_progress` / `max_progress` | `min_episodes`/`max_episodes`, `min_chapters`/`max_chapters` | Просмотрено серий (аниме) или прочитано глав (манга). |
| Личная оценка | `min_score` / `max_score` | — | Ваша оценка записи (0 = без оценки). |
| Общий балл | `min_rating` / `max_rating` | `min_anime_score`/`max_anime_score`, `min_manga_score`/`max_manga_score` | Общий балл тайтла на Шикимори. |

Баллы дробные (например, `8.38`). Чтобы выразить *строго больше 8*, используйте
`min_rating = 8.01`. Баллы запрашиваются только для записей, которые реально
доходят до условия с баллом, так что правила без баллов не делают лишних запросов.

### Пример: прибрать аниме «Запланировано»

```toml
# Всё, что уже начато смотреть, переходит в "watching" (проверяется первым → побеждает).
[[rules]]
media = "anime"
source = "planned"
target = "watching"
min_episodes = 1

# Иначе нетронутые тайтлы с баллом выше 8 уходят в "on_hold".
[[rules]]
media = "anime"
source = "planned"
target = "on_hold"
max_episodes = 0
min_rating = 8.01
```

### Пример: почистить мангу «Брошено»

```toml
# Без оценки, ни одной главы, но с высоким баллом -> отложено.
[[rules]]
media = "manga"
source = "dropped"
target = "on_hold"
max_score = 0
max_chapters = 0
min_rating = 8.01

# Остальные без оценки и без чтения -> запланировано.
[[rules]]
media = "manga"
source = "dropped"
target = "planned"
max_score = 0
max_chapters = 0
```

## Как это работает

- Списки читаются через `GET /api/v2/user_rates` (с `target_type=Anime|Manga`).
  Этот эндпоинт может игнорировать `limit`/`page` и отдавать весь список на
  каждой странице, поэтому клиент прекращает пагинацию, как только страница не
  приносит новых id (а не крутится бесконечно).
- Баллы берутся из `GET /api/animes?ids=...` / `GET /api/mangas?ids=...` пачками по 50.
- Переносы — это `PATCH /api/v2/user_rates/:id` с новым статусом.
- В веб-интерфейсе перенос выполняется фоновой задачей; интерфейс опрашивает её
  прогресс и рисует полосу.
- Все запросы отправляют обязательный `User-Agent` и делают паузу при HTTP 429.

## Безопасность

`config.toml`, `.env*` и `.token.json` содержат секреты и внесены в `.gitignore`.
Никогда не коммитьте их. Если `client_secret` или токен утёк — перевыпустите его
в настройках OAuth-приложения.
